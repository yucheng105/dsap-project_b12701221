# Portrait2Music

## Proposal Report

### 動機與目標
現有的 AI 音樂生成工具（如 Suno, Udio）過於強調「結果的便利性」，使用者僅需輸入文字即可獲得成品，這剝奪了創作過程中的主體性。**Portrait2Music** 的核心價值不在於產出「悅耳」的音樂，而是建立一個**實驗性的聲音轉換工具**。

* **重拾創作主體性**：使用者必須透過親手繪圖來創造聲音，將視覺表現與聽覺實驗結合。
* **在畫布上摸索出自己的聲音**：使用者透過不斷重新繪製、聆聽成品，逐漸創作出有意義的作品。
* **靈感激發器**：為音樂人或一般大眾提供一個非線性的創作媒介，讓聲音的產生變得直觀且具備隨機性。

### 預期功能
系統將提供一個互動式繪圖介面，並支援以下兩種核心映射機制：

1. **座標映射模式 (The Piano Roll Approach)**
   * **時間軸掃描**：掃描線由左至右移動，代表時間序列。
   * **頻率定義**：Y 軸座標反向映射至 MIDI 音高（Note）。
   * **公式參考**： $Note = 127 - (y / canvasHeight * 127)$

2. **內容映射模式 (The Abstract Approach)**
   * **色彩編碼 (HSL/RGB)**：
     * **色相 (Hue)**：決定基本波形（如：紅色=鋸齒波、藍色=正弦波）。
     * **飽和度 (Saturation)**：映射至 Filter Cutoff（濾波器截止頻率）。
     * **亮度 (Lightness)**：控制 Gain（音量增益）。
   * **筆觸屬性**：
     * **線條長度**：定義音符的持續時間 (Duration)。
     * **移動速度**：影響起音 (Attack) 的快慢。
     * **線條粗細**：映射至音色的厚度或失真度 (Detune)。

### 使用技術
本專案將以 **Web App** 形式開發，確保跨平台易用性：
* **畫布實作**：使用 **HTML5 Canvas API**，處理筆跡捕捉與即時像素掃描。
* **音訊引擎**：使用 **Tone.js**，負責 Web Audio API 的封裝，執行振盪器控制、效果器鏈路與時間排程。
* **狀態管理**：使用 JavaScript 物件陣列儲存筆跡數據，確保繪畫過程可追溯且可進行迭代修改。

### 時程規劃
* **Week 1-2：基礎架構搭建**
  * 實作畫布組件（Canvas），支援基本繪圖與清除功能。
  * 整合 Tone.js，建立基本的音訊輸出節點與測試音。
* **Week 3-4：座標映射系統實作**
  * 開發「時間掃描線」邏輯。
  * 完成 Y 軸到 MIDI 音高的映射演算法。
* **Week 5-6：內容映射與進階參數調教**
  * 實作筆跡物件分析（讀取 HSL 色值、筆劃速度、粗細）。
  * 建立顏色與波形、濾波器之間的聯動系統。
* **Week 7：系統迭代與優化**
  * 實作「修改筆跡」功能，讓使用者能針對特定線條進行微調。
  * 優化 UI/UX，提升操作流暢度。
* **Week 8：效能除錯與最終文件**
  * 處理大量像素處理時的延遲問題。
  * 撰寫最終報告與成果展示準備。

### 與課程的關聯
<!-- 你的專題可能涉及哪些資料結構或演算法概念？為什麼？ -->
1. **資料結構 (Array of Objects)**：
   每一道筆觸均存儲為一組包含座標、顏色、時間戳記與壓力的物件序列。如何高效存取與遍歷這些點以進行即時音訊觸發，是資料結構運用的核心。
2. **正規化演算法 (Normalization)**：
   需將畫布的像素範圍轉換為音訊頻率與音量的標準值（0.0 到 1.0），涉及數學轉換與數據標準化邏輯。

## Prototype Report

### 目前進度
<!-- 完成了什麼 -->
* **Canvas 設置完畢**：使用者已能使用三個顏色來進行繪畫。
  * **三種顏色->三種波型**：
    * **紅色**：鋸齒波。聲音較扁、刺。
    * **黃色**：三角波。聲音較鋸齒波溫和一些。
    * **藍色**：正弦波。聲音最圓潤。
  * 使用者可以透過顏色的搭配，控制完成品聲音的音色。
* **筆畫記錄方法**：筆畫資料結構的設計
  * 使用者的每一個筆畫都會成為一個 $currentStroke$ 的物件，紀錄：
    * **顏色**
    * **筆畫寬度**
    * **經過的座標點**
  * 使用者每畫完一筆，現下的 $currentStroke$ 物件就會被 push 入一個名為 $allStrokes$ 的矩陣。
  * 使用者畫完後，程式即可讀取 $allStrokes$ 矩陣，將其轉換為聲音。
  * **完整資料結構如下**：
  ```javascript
  [
    {
      "color": "#ff0000",
      "lineWidth": 5,
      "points": [
        { "x": 10, "y": 10 },
        { "x": 11, "y": 12 },
        { "x": 15, "y": 20 }
        // ... 更多座標點
      ]
    },
    {
      "color": "#0000ff",
      "lineWidth": 2,
      "points": [ ... ]
    }
  ]
  ```
* **To-Sound Feature**：目前已完成 To Sound Feature。做法如下：
  * **x 座標決定時間**：根據 stroke 中的每一個 point 的 x 座標，決定該音的**開始時間**。
  * **y 座標決定音高**：根據 stroke 中的每一個 point 的 y 座標，決定該音的**音高**。
  ```javascript
  // 根據 y 座標值算出一個比率，並將 Canvas 的「往下增加」座標系反轉，變成「往上增加」
  const yPercent = clamp(1 - (point.y / canvasHeight), 0, 1);
  // map to C Major Pentatonic scale
  const noteIndex = Math.floor(yPercent * (CONFIG.SCALE.length - 1)); 
  ```
  * **筆畫寬度決定音量**：根據 stroke 中的 $linewidth$ 來決定音量大小。

### 遇到的困難
<!-- 遇到什麼問題、如何解決或打算如何解決 -->
* **困難 - 轉換音樂過於難聽刺耳**：本打算將筆畫直接轉換為頻率，但結果太難聽。
  * **解決**：將筆畫 mapping 至五聲音階上。五聲音階是一個聽起來很自然的音階。不論音階中的音怎麼組合，樂句聽起來還算合理。
