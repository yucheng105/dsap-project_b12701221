# Portrait2Music

## Proposal Report

### 動機與目標
現有的 AI 音樂生成工具（如 Suno, Udio）過於強調「結果的便利性」，使用者僅需輸入文字即可獲得成品，這剝奪了創作過程中的主體性。**Portrait2Music** 的核心價值不在於產出「悅耳」的音樂，而是建立一個**實驗性的聲音轉換工具**。

* **重拾創作主體性**：使用者必須透過親手繪圖來創造聲音，將視覺表現與聽覺實驗結合。
* **在畫布上摸索出自己的聲音**：使用者透過不斷重新繪製、聆聽成品，逐漸創作出有意義的作品。
* **靈感激發器**：為音樂人或一般大眾提供一個非線性的創作媒介，讓聲音的產生變得直觀且具備隨機性。

### 預期功能
系統將提供一個互動式繪圖介面，並支援以下兩種核心映射機制：

1. **座標映射模式 (The Piano Roll Approach)**
   * **時間軸掃描**：掃描線由左至右移動，代表時間序列。
   * **頻率定義**：Y 軸座標反向映射至 MIDI 音高（Note）。
   * **公式參考**：$Note = 127 - (y / canvasHeight * 127)$。

2. **內容映射模式 (The Abstract Approach)**
   * **色彩編碼 (HSL/RGB)**：
     * **色相 (Hue)**：決定基本波形（如：紅色=鋸齒波、藍色=正弦波）。
     * **飽和度 (Saturation)**：映射至 Filter Cutoff（濾波器截止頻率）。
     * **亮度 (Lightness)**：控制 Gain（音量增益）。
   * **筆觸屬性**：
     * **線條長度**：定義音符的持續時間 (Duration)。
     * **移動速度**：影響起音 (Attack) 的快慢。
     * **線條粗細**：映射至音色的厚度或失真度 (Detune)。

### 使用技術
本專案將以 **Web App** 形式開發，確保跨平台易用性：
* **畫布實作**：使用 **HTML5 Canvas API**，處理筆跡捕捉與即時像素掃描。
* **音訊引擎**：使用 **Tone.js**，負責 Web Audio API 的封裝，執行振盪器控制、效果器鏈路與時間排程。
* **狀態管理**：使用 JavaScript 物件陣列儲存筆跡數據，確保繪畫過程可追溯且可進行迭代修改。

### 時程規劃
* **Week 1-2：基礎架構搭建**
  * 實作畫布組件（Canvas），支援基本繪圖與清除功能。
  * 整合 Tone.js，建立基本的音訊輸出節點與測試音。
* **Week 3-4：座標映射系統實作**
  * 開發「時間掃描線」邏輯。
  * 完成 Y 軸到 MIDI 音高的映射演算法。
* **Week 5-6：內容映射與進階參數調教**
  * 實作筆跡物件分析（讀取 HSL 色值、筆劃速度、粗細）。
  * 建立顏色與波形、濾波器之間的聯動系統。
* **Week 7：系統迭代與優化**
  * 實作「修改筆跡」功能，讓使用者能針對特定線條進行微調。
  * 優化 UI/UX，提升操作流暢度。
* **Week 8：效能除錯與最終文件**
  * 處理大量像素處理時的延遲問題。
  * 撰寫最終報告與成果展示準備。

1. **資料結構 (Array of Objects)**：
   每一道筆觸均存儲為一組包含座標、顏色、時間戳記與壓力的物件序列。如何高效存取與遍歷這些點以進行即時音訊觸發，是資料結構運用的核心。
2. **正規化演算法 (Normalization)**：
   需將畫布的像素範圍轉換為音訊頻率與音量的標準值（0.0 到 1.0），涉及數學轉換與數據標準化邏輯。
